# OA 权限改造说明

这个目录是从原仓库复制出的独立 OA 改造版本，未直接覆盖原始产品目录。

## 已加入能力

- 登录页：内部成员使用邮箱 + PIN 登录。
- 员工端：保留原项目经营驾驶舱、图表、项目列表与 AI 摘要展示。
- 独立后台：管理员可进入成员管理页面。
- 成员管理：新增、编辑、启用、停用成员。
- 项目关联权限：管理员在成员页填写“账号邮箱 / 手机号”和“关联项目”后，员工端只汇总该成员关联项目的合同总额、已回款、待回款和风险。
- 成员通讯录：所有登录用户都可以查看成员列表；股东/管理员可以编辑项目成员绑定，总监和普通成员只读。后续接入飞书通讯录后，新员工入职可同步进入成员库。
- 执行成本表上传：项目可见成员可在项目详情上传月度执行成本表，系统会复用原有 AI 解析流程并归并到该项目；上层人员可在线预览和下载上传文件。
- 后端权限：设置、成员、删除项目等高权限接口仅管理员可操作；项目创建/更新限定管理员、PM、销售。
- 数据兼容：旧 `users` 数据没有邮箱/PIN 时，会自动映射默认账号。

## 默认账号

- 股东：`owner@company.local` / `123456`
- 管理员：`admin@company.local` / `123456`
- 总监：`director@company.local` / `123456`
- 普通员工：`member@company.local` / `123456`
- 项目经理：`pm@company.local` / `123456`
- 销售：`sales@company.local` / `123456`
- 财务：`finance@company.local` / `123456`

上线后请先进入后台成员管理，修改默认 PIN 或替换为 Supabase/Clerk 登录。

## 项目绑定规则

股东或管理员进入“成员”页面后：

1. 填写成员姓名。
2. 填写项目角色，例如 `PM`、`销售`、`财务`、`执行`、`总监`。
3. 填写所属部门，例如 `项目部`、`销售部`。
4. 填写关联项目，支持完整项目名或项目名关键词。
5. 在“账号邮箱 / 手机号”中填写该员工登录邮箱，例如 `member@company.local`。

员工登录后，系统会按登录邮箱、成员姓名、项目负责人/PM/销售字段匹配可见项目。一个 PM 可以绑定多个项目，同一个项目的 PM、销售、财务或执行成员都可以看到该项目的合同金额、回款、待回款、风险提醒和项目详情。

## 分层权限规则

- 股东 `shareholder`：最高权限，可查看全公司所有项目运营，可开通/管理总监、管理员和成员权限，可进入成员、设置、供应商结算等后台页面。
- 管理员 `admin`：系统管理权限，可管理成员、设置、项目和全公司运营数据。
- 总监 `director`：部门运营权限，只汇总自己部门或后台绑定给自己的项目；不能开通总监权限，也不能进入系统设置。
- PM / 销售：项目成员权限，只能看到自己负责或被绑定的项目。
- 普通成员：只看自己参与项目的状态、回款、风险和协作提醒。

## 执行成本表权限

- 执行人员、PM、销售等项目可见成员，可以在自己参与的项目里上传月度执行成本表。
- 上传后不会创建新项目，而是归并到当前项目。
- 文件会进入 AI 解析流程，解析结果用于更新项目执行成本、利润、毛利和供应商费用归属。
- 总监、管理员、股东等上层人员进入该项目详情后，可以查看、预览和下载执行人员上传的成本表。
- 项目无关成员不能向该项目上传成本表。

## 飞书通讯录同步方向

飞书应该作为成员主数据来源。同步后：

- 系统成员列表自动包含飞书在职员工。
- 总监和普通成员可以查看成员列表，但不可编辑。
- 股东/管理员负责开通角色权限和项目绑定。
- 项目成员绑定仍保存在项目中台，例如一个项目绑定 PM、销售、财务、执行成员。
- 员工离职或飞书停用后，后续可同步为系统停用状态。

## 免费优先的后续升级

当前版本使用项目内置轻量账号，适合先验证 OA 权限模型。正式上线后推荐：

1. Supabase Auth 免费档接管登录。
2. `users` 表继续保存业务角色、部门和状态。
3. 前端用 Supabase session 替代本地 PIN 登录。
4. 后端从 Supabase JWT 获取用户身份，再沿用现有 `requireRole` 权限判断。

这样可以继续保持低成本，同时避免长期保存明文 PIN。
